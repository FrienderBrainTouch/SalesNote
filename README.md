# SalesNote UI Design Prototype

영업 특화 AI 코파일럿 플랫폼 **SalesNote**의 웹·모바일 **정적 UI 프로토타입**입니다.  
사업계획서 및 투자 피치용으로, 실제 기능 구현 없이 플로우와 화면 구성을 보여주는 데 목적이 있습니다.

---

## 1. 개요

- **목표**:  
  - 영업 통화 기반 고객 관리/AI 분석 흐름을 한눈에 이해할 수 있는 데모 화면 제공  
  - PC 웹 대시보드와 모바일 앱 주요 플로우를 "디자인만"으로 표현
- **구현 범위**:  
  - 통화 전 브리핑, 통화 중, 통화 후 AI 분석, 웹 대시보드 타임라인 UI  
  - 모든 버튼/입력은 더미이며 **실제 백엔드/AI 연동 없음**

---

## 2. 기술 스택

- **프론트엔드**: React 18 + TypeScript
- **번들러/Dev 서버**: Vite
- **스타일**: 커스텀 CSS (다크 테마, 카드/폰 프레임 스타일)

---

## 3. 주요 화면 구조

### 3.1 PC 웹 대시보드 (`WebLayout`)

- **상단 바**
  - `SalesNote` 로고, 검색창, 알림, 프로필 아바타
- **좌측 고정 패널**
  - 고객 기본 정보 (이름, 소속/직책, 등급, 연락처)
  - 메모 리스트 및 메모 추가 버튼
  - 다음 일정 리스트 (예: 계약 미팅, 팔로업 예정)
  - 통화 통계 (총 통화 횟수, 평균 시간, 감정 트렌드)
- **우측 통화 기록 패널**
  - 상단: "📋 통화 기록" 타이틀 + "새 통화 예약" 버튼
  - 본문: 날짜별 통화 카드가 세로로 꽉 차게 배치된 타임라인
    - **통화 메타**: 일시, 통화 시간
    - **요약 인용문**: 해당 통화의 핵심 한 줄 요약
    - **AI 분석 패널**
      - 요약 (3~5줄)
      - 액션 아이템 체크리스트
      - 감정 분석 바 (긍정/중립/부정 비율)
      - 일정 감지 + 인라인 캘린더 + 일정 등록 UI
      - **전체 텍스트(STT)**: 통화 전문이 담긴 스크롤 박스

### 3.2 모바일 플로우 (`MobileScreens`)

한 화면 안에 4개의 모바일 모형(폰 프레임)으로 구성:

1. **통화 전 브리핑 (수신)**  
   - 수신 전화 배지, 고객 정보, 최근 통화 요약, 메모, 다음 일정, 받기/거절/무음 버튼
2. **앱에서 직접 통화 걸기 (발신)**  
   - 상단 네비(뒤로/이름/편집), 고객 정보, 최근 통화 요약, 고정 메모, 미완료 액션 아이템, "통화 걸기" 버튼
3. **통화 중 화면**  
   - REC 배지 + 타이머, 상대 정보, 빠른 메모 입력, 태그 버튼들(계약/일정/클레임/견적/팔로업), 이전 요약, 통화 종료/음소거/스피커
4. **통화 후 · AI 분석 완료 알림**  
   - AI 분석 완료 배지, 통화 메타, 요약, 감지된 일정 + 캘린더 등록 버튼, 액션 아이템, 전체 분석 보기/확인 버튼

## 4. 실행 방법

```bash
cd SalesNote

# 의존성 설치
npm install

# 개발 서버 실행
npm run dev
```

브라우저에서 `http://localhost:5173` 접속 후,  
PC 웹 대시보드와 모바일 플로우를 동시에 확인할 수 있습니다.

---

## 5. 제약 사항 (사업계획서용 데모 전용)

- 실제 통화 녹음, STT, GPT 호출, 백엔드 API 연동 **전혀 없음**
- 모든 데이터는 컴포넌트 내부에 하드코딩된 **샘플 데이터**
- 클릭/입력은 화면 연출을 위한 것이며, 상태 변경/저장 로직이 없습니다.

실제 서비스 개발 시에는:

- 모바일/웹 분리 프로젝트 구성
- NestJS 백엔드 + PostgreSQL, Redis, S3 연동
- STT/AI 분석 파이프라인 연결
- 사용자 인증/권한, 로그/모니터링

등을 별도 레포지토리 또는 모노레포 구조로 설계하는 것을 전제로 합니다.
