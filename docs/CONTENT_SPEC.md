# SalesNote 화면별 입력 내용 정의

디자인 없이, 각 위치에 **무엇이 들어가야 하는지**만 정의한 문서입니다.

---

## 1. PC 웹 대시보드

### 1.1 상단 바
| 위치 | 입력 내용 |
|------|-----------|
| 로고 영역 | 서비스명(영문 또는 국문) |
| 구분선 오른쪽 | 서비스 부제 또는 한글명 (예: 영업 노트) |
| 검색창 placeholder | 고객/회원 검색 시 사용할 안내 문구 (예: Q 이름 검색, 고객명·메모·키워드 검색) |
| 우측 버튼 | 알림, 설정, 내보내기 등 글로벌 액션 중 택 1 또는 복수 |
| 아바타 | 로그인 사용자 이니셜 또는 프로필 이미지 |

### 1.2 좌측 패널 (고객 프로필 고정)
| 위치 | 입력 내용 |
|------|-----------|
| 아바타 | 해당 고객(회원) 이름 이니셜 또는 프로필 이미지 |
| 이름 | 고객(회원) 성함 |
| 소속/직급 한 줄 | 소속 회사·부서명 (예: 한솔제지 영업1팀) |
| 직급/등급 한 줄 | 직책·직급 및 고객 등급(예: 차장, VIP) — 소속과 줄바꿈 구분 권장 |
| 연락처 | 대표 전화번호 (또는 이메일 등) |
| 메모 섹션 제목 | 고정 라벨 (예: 메모) |
| 메모 목록 | 사용자가 입력·핀 고정한 메모 문장 목록 (항목별 한 줄) |
| 메모 추가 버튼 | 고정 라벨 (예: + 메모 추가) |
| 다음 일정 섹션 제목 | 고정 라벨 (예: 다음 일정) |
| 일정 목록 각 항목 | 일정 날짜·시간 + 일정 제목(유형) (예: 2024.02.22 (목) 오전 10:00 / 계약 미팅) |
| 통화 통계 섹션 제목 | 고정 라벨 (예: 통화 통계) |
| 통계 행 | 라벨 + 수치 (총 통화 N회, 평균 통화 시간, 감정 추이 등) |

### 1.3 우측 패널 (통화 기록)
| 위치 | 입력 내용 |
|------|-----------|
| 상단 타이틀 | 고정 라벨 (예: 통화 기록) |
| 상단 버튼 | 기록과 연관된 주 액션 (예: 새 통화 예약) |
| 통화 카드 — 1행 | 해당 통화 일시(날짜·요일·시각) + 통화 시간(길이) |
| 통화 카드 — 요약 인용 | 해당 통화의 한 줄 요약 또는 핵심 문장 |
| 통화 정리(AI) 블록 제목 | 고정 라벨 (예: 통화 정리, AI 분석) |
| 요약 문단 | 해당 통화 내용 3~5줄 요약 |
| 해야 할 일 제목 | 고정 라벨 (예: 해야 할 일, 액션 아이템) |
| 해야 할 일 목록 | 체크리스트 형태의 할 일 문장 (미완료 시 ☐) |
| 감정 흐름 제목 | 고정 라벨 (예: 감정 흐름, 감정 분석) |
| 감정 흐름 | 긍정·중립·부정 비율 또는 짧은 설명 문구 |
| 나온 일정 제목 | 고정 라벨 (예: 나온 일정, 일정 감지) |
| 나온 일정 설명 | 통화에서 추출한 일정 관련 한 줄 문장 |
| 인라인 캘린더 | 해당 월의 달력 + 선택/강조 날짜 |
| 일정 등록 확인 블록 제목 | 제안 문장 (예: YYYY년 M월 D일 (요일)에 등록할까요?) |
| 일정 등록 — 제목 필드 | 자동 채워진 제목(예: 계약 미팅 - 고객명) |
| 일정 등록 — 시간 필드 | 자동 채워진 시간(예: 오전 10:00) |
| 일정 등록 버튼 | 고정 라벨 (예: 일정 등록, 나중에) |
| 전체 텍스트(STT) 제목 | 고정 라벨 (예: 전체 텍스트 (STT)) |
| STT 본문 | 화자 구분이 있는 통화 전문 텍스트 (영업/고객 또는 이름 라벨) |

## 2. 모바일 앱

### 2.1 통화 전 브리핑 (수신)
| 위치 | 입력 내용 |
|------|-----------|
| 상단 배지 | 수신 상태 안내 (예: 수신 전화) |
| 아바타 | 고객 이름 이니셜 또는 프로필 이미지 |
| 이름 | 고객 성함 |
| 소속(1줄) | 회사·부서명 (줄바꿈 권장) |
| 직급/등급(1줄) | 직책·직급 및 VIP 등 등급 (줄바꿈 권장) |
| 연락처 | 전화번호 |
| 최근 통화 요약 제목 | 고정 라벨 (예: 최근 통화 요약) |
| 최근 통화 요약 목록 | 날짜 + 해당 통화 한 줄 요약 (여러 건) |
| 메모 제목 | 고정 라벨 (예: 메모) |
| 메모 목록 | 고정/핀 메모 문장 목록 |
| 다음 예정 일정 제목 | 고정 라벨 (예: 다음 예정 일정) |
| 다음 일정 1개 | 날짜·시간 + 일정 제목 (한 줄 또는 pill 형태) |
| 받기/거절 버튼 | 고정 라벨 (받기, 거절) |
| 무음으로 받기 | 고정 라벨 (예: 무음으로 받기) |

### 2.2 통화 전 브리핑 (발신 — 앱에서 통화 걸기)
| 위치 | 입력 내용 |
|------|-----------|
| 상단 좌 | 뒤로가기 등 네비 라벨 |
| 상단 중 | 해당 고객 이름 |
| 상단 우 | 편집 등 보조 액션 라벨 |
| 아바타 | 고객 이니셜 또는 이미지 |
| 이름 | 고객 성함 |
| 소속(1줄) / 직급(1줄) | 회사·부서 / 직책·등급 (줄바꿈 권장) |
| 최근 통화(날짜) 제목 | 고정 라벨 (예: 최근 통화 (MM.DD)) |
| 최근 통화 요약 문단 | 해당 통화 한 줄 요약 |
| 고정 메모 제목 | 고정 라벨 (예: 고정 메모) |
| 고정 메모 목록 | 메모 문장 목록 |
| 미완료 액션 제목 | 고정 라벨 (예: 미완료 액션, 미완료 액션 아이템) |
| 미완료 액션 목록 | 체크리스트 문장 (☐) |
| 통화 걸기 버튼 | 고정 라벨 (예: 통화 걸기) |
| 하단 안내 문구 | 통화 시 녹음 등 안내 (예: 통화 시작 시 자동 녹음됩니다) |

### 2.3 통화 중 화면
| 위치 | 입력 내용 |
|------|-----------|
| 상단 배지 | 통화 중 상태 + REC + 경과 시간 (예: 통화 중 REC 00:03:42) |
| 아바타 | 상대방 이니셜 또는 이미지 |
| 이름 | 상대방 성함 |
| 소속(1줄) | 회사·부서명 (줄바꿈 권장) |
| 직급(1줄) | 직책·직급 (줄바꿈 권장) |
| 연락처 | 전화번호 |
| 빠른 메모 제목 | 고정 라벨 (예: 빠른 메모) |
| 메모 입력 placeholder | 입력 유도 문구 (예: 메모 적기...) |
| 태그 영역 | 태그 목록 (예: #계약, #일정, #클레임, #견적, #팔로업) |
| 참고(이전 요약) 제목 | 고정 라벨 (예: 참고 (이전 요약)) |
| 참고 문단 | 이전 통화 요약 또는 브리핑 핵심 한 줄 |
| 통화 종료 버튼 | 고정 라벨 (예: 통화 종료) |
| 음소거/스피커 버튼 | 고정 라벨 (음소거, 스피커) |

### 2.4 통화 후 (분석 완료 알림)
| 위치 | 입력 내용 |
|------|-----------|
| 상단 배지 | 완료 상태 (예: 분석 완료) |
| 통화 메타 | 고객명 + 통화 시간 (예: 김준영 · 8분 23초) |
| 요약 제목 | 고정 라벨 (예: 요약) |
| 요약 문단 | 해당 통화 2~4줄 요약 |
| 나온 일정 제목 | 고정 라벨 (예: 나온 일정, 감지된 일정) |
| 나온 일정 한 줄 | 날짜·시간 + 일정 제목 |
| 캘린더 등록 버튼 | 고정 라벨 (예: 캘린더에 넣기) |
| 해야 할 일 제목 | 고정 라벨 (예: 해야 할 일, 액션 아이템) |
| 해야 할 일 목록 | 체크리스트 문장 (☐) |
| 전체 보기 버튼 | 고정 라벨 (예: 전체 보기) |
| 확인 버튼 | 고정 라벨 (예: 확인) |

## 3. 공통 규칙

- **고정 라벨**: 위 표에 적힌 대로 서비스 전역에서 동일 문구 사용 권장.
- **고객(회원) 정보**: 이름, 소속, 직급, 연락처는 회원(고객) 마스터 데이터에서 조회·표시.
- **소속/직급 줄바꿈**: 모바일에서는 소속(회사·부서) 1줄, 직급·등급 1줄로 나누어 표시 권장.
- **날짜·시간 형식**: 서비스 내 공통 포맷 정의 후 통일 (예: 2024.02.22 (목) 오전 10:00).
- **통화 요약·STT·액션·일정**: 서버/AI 분석 결과 또는 사용자 입력값을 해당 위치에 매핑.
